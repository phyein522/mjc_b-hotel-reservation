## 2. 세부 설계 및 구현 내용

### 2.1. DTO 설계 (추가될 응답 규격)

#### 1) 채널별 매출/예약 비중 DTO (`ChannelShareDto`)
```java
@Schema(description = "채널별 예약 및 매출 비중")
@Data
@Builder
public class ChannelShareDto {
    @Schema(description = "채널 ID", example = "1")
    private Long channelId;
    @Schema(description = "채널명 (예: 야놀자, 아고다, 직접예약)", example = "야놀자")
    private String channelName;
    @Schema(description = "해당 채널 예약 건수", example = "15")
    private Long bookingCount;
    @Schema(description = "예약 건수 비중 (%)", example = "50.0")
    private Double bookingShareRate;
    @Schema(description = "해당 채널 발생 매출액", example = "2250000.00")
    private BigDecimal revenue;
    @Schema(description = "매출 비중 (%)", example = "45.0")
    private Double revenueShareRate;
}
```

#### 2) 매출 상위 예약 DTO (`TopBookingDto`)
```java
@Schema(description = "매출 상위 예약 상세 정보")
@Data
@Builder
public class TopBookingDto {
    @Schema(description = "순위", example = "1")
    private Integer rank;
    @Schema(description = "예약자 성함", example = "홍길동")
    private String guestName;
    @Schema(description = "이용 객실명", example = "103호 (스위트 룸)")
    private String roomName;
    @Schema(description = "총 결제금액 (매출)", example = "900000.00")
    private BigDecimal amount;
    @Schema(description = "총 숙박 수 (박수)", example = "3")
    private Integer nights;
}
```

### 2.2. MyBatis SQL 설계 (초안)

#### 1) 채널별 예약 및 매출 비중 쿼리
```sql
SELECT 
    b.channel_id as channelId,
    COALESCE(c.채널명, '기타') as channelName,
    COUNT(b.booking_id) as bookingCount,
    -- 예약비율 = (해당 채널 건수 / 전체 채널 건수) * 100
    ROUND((COUNT(b.booking_id) * 100.0) / SUM(COUNT(b.booking_id)) OVER(), 2) as bookingShareRate,
    SUM(p.amount) as revenue,
    -- 매출비율 = (해당 채널 매출 / 전체 채널 매출) * 100
    ROUND((SUM(p.amount) * 100.0) / SUM(SUM(p.amount)) OVER(), 2) as revenueShareRate
FROM bookings b
JOIN payments p ON b.booking_id = p.booking_id
LEFT JOIN channels c ON b.channel_id = c.채널ID
WHERE b.hotel_id = :hotelId
  AND p.payment_status = 'PAID'
  AND p.paid_at BETWEEN :startDate AND :endDate
GROUP BY b.channel_id, c.채널명;
```

#### 2) 매출액 기준 상위 1~5위 예약 조회 쿼리
```sql
SELECT 
    CONCAT(b.guest_last_name, b.guest_first_name) as guestName,
    CONCAT(r.room_number, '호 (', r.name, ')') as roomName,
    p.amount as amount,
    b.nights as nights
FROM bookings b
JOIN payments p ON b.booking_id = p.booking_id
JOIN rooms r ON b.room_id = r.room_id
WHERE b.hotel_id = :hotelId
  AND p.payment_status = 'PAID'
  AND p.paid_at BETWEEN :startDate AND :endDate
ORDER BY p.amount DESC, b.nights DESC
LIMIT 5;
```

## 3. 핵심 마일스톤 (성공 기준)
1. **시작 후 50분 시점**: 두 통계 쿼리가 XML 작성을 완료하고 컴파일에 에러가 없음.
2. **시작 후 1시간 40분 시점**: JUnit 테스트 구동 시 채널 비중 합산이 100%에 정확히 도달하고, 매출 상위 1위의 결제금액이 의도된 수치(`900,000`원)로 어서트 통과됨.
3. **시작 후 2시간 시점**: Swagger 콘솔을 통해 UI용 신규 지표 데이터 배열 2개가 정상 리턴되는 것을 확인.
