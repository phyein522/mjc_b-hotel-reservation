## 2. 주차별 상세 계획 (Detailed Weekly Tasks)

### 🚀 1주차 : 데이터베이스 연동 & 기초 인프라 구축
* **목표**: `hotel_erd (5).sql`의 물리적 테이블을 데이터베이스에 구축하고, Spring Boot 프로젝트에 JPA 엔티티로 매핑하여 기본 영속성 레이어를 완성합니다.
* **상세 작업 태스크**:
  1. **DB 스키마 생성**: 로컬 및 개발 Mysql에 ERD SQL을 반영하여 테이블 생성.
  2. **JPA Entity 맵핑**: 
     * `Hotels` (호텔), `Rooms` (객실), `Bookings` (예약), `Payments` (결제), `Users` (회원) 엔티티 생성.
     * 엔티티 간의 관계 설정 및 영속성 전이(Cascade) 검토.
  3. **더미 데이터 구축**:
     * 전월 및 전년 동월 지표 비교, 그리고 7개월간의 매출 트렌드를 정상적으로 조회하기 위해 최소 **1년 6개월 치의 결제 및 예약 더미 데이터 SQL**을 작성하여 DB에 주입.
  4. **JPA Repository 생성**: 각 엔티티의 기본 CRUD를 위한 `JpaRepository` 정의.

---

### 📊 2주차 : MyBatis 통계 쿼리 및 서비스 비즈니스 로직 개발
* **목표**: 대시보드에 노출될 복잡한 집계 수치들을 연산하는 MyBatis Mapper를 설계하고, 이를 호출하는 비즈니스 서비스 레이어를 구현합니다.
* **상세 작업 태스크**:
  1. **통계 쿼리 (MyBatis) 작성**:
     * `SalesAnalysisMapper.xml` 생성 및 복잡한 SQL 작성.
     * 특정 월의 **총 매출액**, **예약 건수**, **점유율**, **평균 객단가(ADR)** 집계.
     * **고객 재방문율** 및 **VIP 고객 재방문율** 추출 쿼리 설계 (조인 및 서브쿼리 활용).
     * 조회월 기준 **직전 7개월간의 월별 매출 트렌드** 조회 쿼리.
  2. **SalesAnalysisService 구현**:
     * 조회 파라미터(`yearMonth`) 유효성 검사 및 파싱 (값이 없을 경우 현재 달 설정).
     * 전월 대비 성장률(%) 연산 처리 (`(이번달 - 전월) / 전월 * 100`).
     * 매출 및 점유율 목표치 테이블이 설계상 없으므로, 설정 값(또는 목표 관리용 임시 데이터 테이블)을 비교하는 로직 구성.
  3. **비즈니스 로직 검증 (단위 테스트)**:
     * 더미 데이터를 바탕으로 구현된 쿼리와 서비스 로직의 수치 정확도를 단위 테스트(`JUnit`)로 검증.

---

### 🔑 3주차 : REST API 컨트롤러 설계 & Security / Swagger 연동
* **목표**: 프론트엔드 연동을 위한 REST API 규격을 확정하고, JWT 인증 필터를 적용하여 대시보드 보안을 강화하며 Swagger UI를 통해 문서를 제공합니다.
* **상세 작업 태스크**:
  1. **DTO 설계 및 명세**:
     * UI 컴포넌트 렌더링에 적합하도록 정렬된 필드들로 구성된 응답 DTO (`SalesDashboardResponse`) 정의.
  2. **SalesAnalysisController 작성**:
     * `GET /api/sales/dashboard` 엔드포인트 구현.
  3. **JWT Security 설정 적용**:
     * 기존 `SecurityConfig` 파일의 인가 규칙을 바탕으로 `/api/sales/**` 보안 검토.
     * 대시보드는 비즈니스 데이터이므로 일반 사용자가 아닌 **호텔 매니저(`HOTEL_MANAGER`) 및 관리자(`ADMIN`)** 권한을 가진 주체만 JWT를 통해 접근할 수 있도록 롤(Role) 기반의 인가(`@PreAuthorize`) 처리 적용.
  4. **Swagger 문서화**:
     * 컨트롤러와 DTO 객체에 OpenAPI 3 어노테이션 (`@Tag`, `@Operation`, `@Schema`) 추가로 API 설명 작성.

---

### 🛠️ 4주차 : 프론트엔드 연동 지원, 성능 최적화 및 최종 검증
* **목표**: 프론트엔드(UI) 웹뷰 및 차트 연동을 지원하고 대용량 데이터 조회 시의 쿼리 성능 문제를 튜닝하여 프로젝트를 최종 마무리합니다.
* **상세 작업 태스크**:
  1. **프론트엔드 연동 및 CORS 지원**:
     * 프론트엔드 서버와 원활한 통신을 위해 CORS 설정 구성.
     * 막대그래프(Recharts/Chart.js) 컴포넌트에 연동 시 데이터 순서 및 날짜 포맷팅 최종 조정.
  2. **쿼리 성능 최적화 (인덱싱)**:
     * 통계 집계 시 자주 필터링되는 날짜 필드(`bookings.check_in_date`, `payments.paid_at`)와 외래키(`hotel_id`) 칼럼에 대한 데이터베이스 인덱싱 적용 계획 및 인덱스 튜닝.
  3. **예외 처리 & 에러 로깅**:
     * 존재하지 않는 호텔 조회, 잘못된 날짜 형식 유입 시 적절한 에러 메시지와 HTTP 상태 코드를 반환하도록 전역 예외 처리기 (`advice/GlobalExceptionHandler`) 연동.
  4. **최종 통합 테스트 및 배포 준비**:
     * 전체 시나리오를 바탕으로 API 통합 테스트 실행 및 릴리즈 빌드 검증.
