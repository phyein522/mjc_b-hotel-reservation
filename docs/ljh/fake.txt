5. API 동작 모드 구분 (Stubbed vs Real SQL)
--------------------------------------------------------------------------------
동일한 API 컨트롤러(/api/sales/**)를 사용하되, 설정(Profile)에 따라 다르게 작동합니다.

* [Stub API 모드] (spring.profiles.active=stub)
  - 대상 구현체: SalesAnalysisServiceStub.java
  - 동작 방식: 데이터베이스 쿼리를 일절 수행하지 않으며, 코드 내부에서 생성한 완전한 형태의 모의(Mock) 데이터 목록을 즉시 리턴합니다.
  - 대상 API 및 반환값:
    1) GET /api/sales/dashboard: 모의 매출액, 예약수, 점유율 등 종합 Metrics 및 TOP 3 예약 내역 반환
    2) GET /api/sales/monthly: 8개월 분량의 모의 월 매출 목록 반환
    3) GET /api/sales/channels: 아고다, 야놀자, 직접예약의 모의 점유율 반환
    4) GET /api/sales/top-bookings: 3건의 모의 우수 예약 목록 반환
    5) GET /api/sales/rooms: 객실 유형별 모의 매출 통계 반환

* [Real SQL API 모드] (spring.profiles.active=default 또는 stub 외의 값)
  - 대상 구현체: SalesAnalysisServiceImpl.java
  - 동작 방식: MyBatis XML 매퍼의 SELECT 쿼리를 실제 가동하여 DB 테이블의 원시 데이터를 집계하고 자바 단에서 수학적 연산을 가쳐 최종 DTO를 반환합니다.
  - 대상 API 및 내부 SQL 연동 매핑:
    1) GET /api/sales/dashboard: getBasicMetrics, getRoomTypeRevenue, getChannelShares, getTopBookings, getReturningGuestRate, getVipReturningGuestRate 쿼리 결과를 합산하고, 자바 코드에서 ADR, 점유율 및 전월 대비 증감률을 연산하여 반환
    2) GET /api/sales/monthly: getMonthlyRevenueTrend 쿼리 연동
    3) GET /api/sales/channels: getChannelShares 쿼리 연동
    4) GET /api/sales/top-bookings: getTopBookings 쿼리 연동
