# 딥인터뷰 요약: review-crud

## 사용자 요청
예약관리프로그램의 리뷰 부분 CRUD를 구현한다. 기존 파일과 패키지 구조를 유지하고, DB는 `application.yaml`에 명시된 설정을 사용한다. H2는 사용하지 않고, 프론트엔드는 만들지 않는다. DB 구조는 `HELP.md`를 참고한다.

## 확인된 답변
- `student` 패키지는 실제 프로젝트 도메인이 아니라 앞으로 작성할 코드 구조의 예시로만 참고한다.
- Figma 예약관리프로그램 화면 정의는 실제 구현 대상이 아니라 리뷰 기능 구성을 참고하기 위한 자료다.
- 리뷰 CRUD에는 `reviews_photo`, `reviews_tags`, `reivews_rating` 같은 사진, 태그, 카테고리별 평점까지 포함한다.

## 구현 방향
`com.mjc.hotel.review` 패키지에 Entity, Repository, DTO, Service, Controller를 추가해 리뷰 본문과 하위 데이터를 함께 CRUD한다.
