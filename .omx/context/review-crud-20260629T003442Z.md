# 딥인터뷰 컨텍스트 스냅샷: review-crud

## 작업 내용
예약관리프로그램의 리뷰 부분에 대한 백엔드 CRUD를 구현한다.

## 원하는 결과
리뷰 관련 CRUD API를 만든다. 프론트엔드는 만들지 않는다.

## 사용자가 지정한 구현 방향
- 현재 생성되어 있는 파일과 패키지 구조를 유지한다.
- 현재 프로젝트에 이미 포함된 패키지와 의존성을 유지한다.
- 데이터베이스는 `hotel/src/main/resources/application.yaml`에 명시된 DB 설정을 사용한다.
- H2는 사용하지 않는다.
- DB 구조는 `hotel/HELP.md`를 참고한다.
- Figma의 예약관리프로그램 화면 정의는 실제 프론트엔드 구현 대상이 아니라, 리뷰 부분이 어떻게 구성될지 참고하기 위한 자료로만 사용한다.

## 의도 가설
사용자는 기존 Spring Boot 호텔 예약 프로젝트의 구조를 유지하면서, `HELP.md`에 있는 리뷰 관련 DB 구조를 바탕으로 리뷰 CRUD 백엔드 기능을 추가하려는 것으로 보인다.

## 확인된 사실 / 근거
- 프로젝트 루트 아래 `hotel` 디렉터리에 Gradle 기반 Spring Boot 프로젝트가 있다.
- `hotel/build.gradle`에는 Spring Boot Web MVC, Spring Data JPA, MyBatis, MariaDB driver, Security, Swagger, Lombok이 포함되어 있다.
- `hotel/src/main/resources/application.yaml`은 MariaDB를 사용하도록 설정되어 있다.
  - URL: `jdbc:mariadb://localhost:33306/hotel`
  - username: `root`
  - password: `1234`
  - driver: `org.mariadb.jdbc.Driver`
- 현재 의존성 목록에는 H2가 없다.
- `student` 패키지는 앞으로 작성할 코드 구조의 예시로만 참고한다.
- `student` 패키지는 실제 프로젝트 도메인과 연관이 없고, 실제 기능 구현에 사용되는 패키지가 아니다.
- `hotel/HELP.md`에는 리뷰 관련 테이블로 `reviews`, `reviews_tags`, `reviews_photo`, `rating_categories`, `reivews_rating`, `tag_type`, `check_trip_type` 등이 포함되어 있다.
- `HELP.md`의 리뷰 관련 컬럼명 일부는 인코딩이 깨진 한글로 보이며, 현재 Java 코드에서는 주로 영문 식별자를 사용한다.

## 제약 조건
- H2를 추가하거나 사용하지 않는다.
- 프론트엔드는 구현하지 않는다.
- 기존 패키지와 의존성 방향을 최대한 유지한다.
- `student` 패키지는 참고만 하고 실제 리뷰 기능 구현에는 연결하지 않는다.
- 기존 사용자 변경사항은 되돌리지 않는다. 작업 전 `git status`에서 `hotel/src/main/java/com/mjc/hotel/user/entity/User.java`가 이미 수정된 상태였다.

## 확정된 범위
- 리뷰 CRUD에는 메인 `reviews` 테이블뿐 아니라 `reviews_photo`, `reviews_tags`, `reivews_rating`에 해당하는 사진, 태그, 카테고리별 평점도 함께 포함한다.
- 리뷰 생성, 조회, 수정, 삭제 시 사진, 태그, 카테고리별 평점 데이터를 함께 다룬다.

## 아직 남은 확인 사항
- 현재 프로젝트에 명확한 REST API 경로 규칙이 자리 잡혀 있지 않다. 기존 컨트롤러는 `/mapperUsers`, `/repositoryStudents`처럼 단순한 루트 경로를 사용하고 있다.

## 결정 경계
- 사용자가 사진, 태그, 카테고리별 평점까지 포함한다고 확정했다.
- 에이전트는 DTO, Entity, Repository, Service, Controller 구조를 정해서 리뷰 집합 CRUD를 구현할 수 있다.

## 예상 코드 작업 위치
- `hotel/src/main/java/com/mjc/hotel/review/**`
- 필요 시 `hotel/src/test/java/com/mjc/hotel/review/**`
- 검증 과정에서 컴파일 또는 런타임 문제가 드러나지 않는 한 `build.gradle`과 `application.yaml`은 변경하지 않는다.

## 확인한 문서 / 규칙 / 코드
- 사용자 프롬프트에 포함된 AGENTS.md 지침
- 루트 `README.md`
- `hotel/HELP.md`
- `hotel/build.gradle`
- `hotel/src/main/resources/application.yaml`
- 기존 `user` 패키지
- 구조 참고용 `student` 패키지
- 프로젝트 내부에 별도 `AGENTS.md` 파일은 발견되지 않았다.

## 용어 / 문서와 코드 사이의 충돌
- `HELP.md`의 리뷰 테이블 컬럼명 일부가 깨진 한글로 표시된다.
- 실제 Java 코드에서는 영문 필드명과 패키지명을 사용하는 편이므로, 리뷰 기능도 Java 코드에서는 영문 식별자를 사용하는 방향이 자연스럽다.
- `student` 패키지는 구조 예시일 뿐 실제 도메인 기능으로 취급하지 않는다.
